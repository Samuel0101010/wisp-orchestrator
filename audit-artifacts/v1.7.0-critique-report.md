# v1.7.0 Design Critique — Agent Harness

Read-only audit. Visual evidence in `audit-artifacts/screenshots/v1.7.0-baseline-*.png`. Token system in `apps/dashboard-web/src/styles/tokens-primitive.css` and `tokens-component.css`. The critique uses the impeccable design laws (absolute bans on side-stripes, hero-metric template, identical card grids) and ui-ux-pro-max heuristics (rhythm, density, hierarchy, state).

Screenshots referenced: `home-{dark,light}`, `chat-{dark,light}`, `agents-{dark,light}`, `skills-{dark,light}`, `workers-{dark,light}`, `insights-dark`, `goap-dark`, `prompt-bundles-dark`, `project-detail-{dark,light}`, `project-teams-dark`, `project-plan-{dark,light}`, `project-run-{dark,light}`.

## Headline findings

1. **Two absolute-ban violations are live and pervasive.** KPI tiles on `/` use a left side-stripe via `before:` pseudo (`apps/dashboard-web/src/components/home/KpiTile.tsx:53` — `before:left-0 before:top-0 before:h-full before:w-0.5`). Run kanban task cards use the same pattern with an absolute div (`apps/dashboard-web/src/routes/RunView.tsx:200-204` — `absolute left-0 top-0 h-full w-1 style={roleStripeStyle(...)}`). Plan canvas nodes add a 4px colored top stripe (`apps/dashboard-web/src/components/plan/PlanCanvas.tsx:47` — `<div style={{ background: accent, height: 4 }} />`). These are the single biggest "AI slop" tell across the product.
2. **The "hero metric template" is the home page.** Four identical-density KPI cards in a row, big numbers, small caption, color accent — `home-dark.png`. Textbook SaaS cliché. The cards are visually a family but the family is generic.
3. **Brand presence is anemic.** The sidebar header is "Agent Harness" body-sm + a generic shadcn `Badge` reading "v1.6.0" (`Sidebar.tsx:130-136`). No logomark, no monospaced version, no product chroma. The product has Linear-class motion tokens defined (`--ease-smooth`, `--duration-base`) but the chrome reads as a generic admin template.
4. **Spacing is uniform to the point of monotony.** Cards on `/`, `/agents`, `/skills`, `/projects/:id` all share the same `p-4`/`p-5` rounded-lg border-card shell. No tier of importance is expressed through rhythm. impeccable: "vary spacing for rhythm. Same padding everywhere is monotony."
5. **Empty states are placeholders, not designed.** `project-run-dark.png` Pending / Running / Verifying columns show centered grey "empty" text. `prompt-bundles-dark.png` has one row, then an ocean of empty card area. `goap-dark.png` says "Submit a problem to see the plan." with nothing else on a 1440-wide canvas.
6. **Status pills use 7+ different shapes.** Compare `LOCKED` (sidebar, small grey rounded-md), `FAILED` (red dot pill), `FAILED (Failure)` (red badge in run header), `seed/HAIKU/SONNET` (Skills page — pastel, mixed casing), `enabled/disabled` (workers, lowercase). Casing, weight, and shape vary per screen.

## P0 (must-fix for "extreme quality")

### 1. Eliminate left side-stripe on KPI tiles

- File: `apps/dashboard-web/src/components/home/KpiTile.tsx` (lines 25-31, 52-56)
- Current: `before:left-0 before:top-0 before:h-full before:w-0.5` with `toneToBg` mapping `info/success/warning/destructive → bg-{tone}`.
- Proposed: Drop the entire `before:*` accent. Replace with a small **leading icon badge** in the top-left (existing `icon` prop placement moves to a 28×28 rounded-md soft-tinted square: `bg-{tone}/10 text-{tone}`) and a `--shadow-xs` at rest, `--shadow-sm` on hover. Tone communicates via the icon chip, not a stripe.
- Impact: `/` Mission Control.
- Effort: S.

### 2. Eliminate left side-stripe on run task cards

- File: `apps/dashboard-web/src/routes/RunView.tsx` (lines 195-256)
- Current: absolute role-color stripe (`absolute left-0 top-0 h-full w-1 style={roleStripeStyle(task.role)}`) plus `pl-2` everywhere to clear it.
- Proposed: Remove the stripe div. Replace with a small role token at the top: `<span className="inline-flex items-center gap-1.5"><span className="size-1.5 rounded-full" style={{ background: roleHsl(task.role) }} /><span className="text-xs2 font-medium uppercase tracking-wider text-muted-foreground">{role}</span></span>`. Drop the existing right-aligned `Badge variant="outline"` for role — redundant. Drop the `pl-2` clears throughout. Status (pending/running/done/failed) becomes the **card-level signal** via a subtle 1px ring color on the running/failed states only: `data-status=running:ring-1 ring-info/40`, `data-status=failed:ring-1 ring-destructive/40`.
- Impact: `/projects/:id/run/:runId` — the product's "wow" surface.
- Effort: M.

### 3. Eliminate top stripe on plan-canvas nodes; rework role chip

- File: `apps/dashboard-web/src/components/plan/PlanCanvas.tsx` (lines 38-65)
- Current: 4px colored top bar (`height: 4`) + filled white-on-color role chip (`background: accent, color: white`).
- Proposed: Drop the 4px bar entirely. The role chip becomes a low-chroma tinted pill, not a filled saturated block: `background: hsl(${roleHue} 70% 95% / 0.6); color: hsl(${roleHue} 60% 30%); border: 1px solid hsl(${roleHue} 60% 30% / 0.25)` (dark mode mirror at L=18%/85%). This still ties node identity to role but stops shouting. Reflow the title to be the strongest element (`text-sm font-semibold` stays, but role pill recedes to `text-2xs font-medium`).
- Impact: `/projects/:id/plan` — high-visibility editor.
- Effort: S.

### 4. Redesign the sidebar brand block

- File: `apps/dashboard-web/src/components/layout/Sidebar.tsx` (lines 130-137)
- Current: Plain `text-sm font-semibold` "Agent Harness" stacked above a shadcn `Badge variant="secondary"` reading "v1.6.0". Reads as placeholder.
- Proposed: Replace with a horizontal block — left side: 24×24 SVG logomark (geometric wisp/hex shape, single solid `--color-foreground`); right side: `text-sm font-semibold tracking-tight` "Agent Harness" with `text-2xs font-mono tabular-nums text-muted-foreground/70` "v1.6.0" directly underneath the wordmark (no pill). Remove `<Badge>`. Add `padding-block: var(--space-4)` and a `--shadow-xs`-strength bottom hairline (use existing `<Separator />`).
- Impact: Every route — sidebar is on every screen.
- Effort: S (logomark is one SVG file).

### 5. Replace the four-card KPI hero with a denser, opinionated header

- File: `apps/dashboard-web/src/routes/Home.tsx` (around the KPI grid)
- Current: Four 1:1 KpiTile cards: Active runs / Tokens 7d / Success rate 7d / Avg duration 7d. Identical density, identical visual weight.
- Proposed: Single **horizontal metric bar** at top — 4 metrics laid out inline (not as cards) with thin vertical dividers between them. Format: stacked label-eyebrow (`text-2xs uppercase tracking-widest text-muted-foreground`) + large `tabular-nums font-semibold text-2xl` value + delta arrow inline. No card chrome, no per-metric border, no per-metric accent stripe. The headline metric (Active runs, since this is mission control) gets `text-3xl` and a soft `bg-success/5` on the entire row when > 0. Save vertical space → Token throughput chart can grow.
- Impact: `/` Mission Control — first impression of the product.
- Effort: M.

### 6. Design the empty states

- Files: `apps/dashboard-web/src/routes/RunView.tsx` (kanban empty), `routes/PromptBundles.tsx`, `routes/Goap.tsx`, `routes/Workers.tsx` (when 0)
- Current: Centered grey "empty" string with no affordance.
- Proposed: Per-column-empty in run kanban: muted icon (Pause/Play/Check/X from lucide matching the column semantic) + one-line caption + thin `border-dashed border-border/40`. Per-page-empty (prompt bundles, goap, workers-empty-state) gets a centered illustration block: 64×64 lucide icon in `text-muted-foreground/30`, a short heading (`text-base font-medium`), a one-sentence helper (`text-sm text-muted-foreground max-w-prose`), and a single primary CTA. Reuse this as `<EmptyState icon label description action />` in `components/ui/empty-state.tsx`.
- Impact: every empty surface in the product.
- Effort: M.

### 7. Standardize status pills

- Files: `apps/dashboard-web/src/components/ui/badge.tsx`, callers across `Workers.tsx`, `Skills.tsx`, `RunView.tsx`, `ProjectDetail.tsx`
- Current: Casing inconsistent (`FAILED` vs `enabled` vs `Locked` vs `seed`), shapes inconsistent (rounded-md vs rounded-full), dot-prefix only on some.
- Proposed: One status-pill component with three variants: `solid` (high-importance terminal states — Failed, Locked, Done), `soft` (in-progress — Running, Verifying — uses `bg-{tone}/12 text-{tone}` with a 6px leading pulsing dot for live states), `outline` (taxonomy tags — seed/user/plugin, sonnet/haiku/opus). All UPPERCASE 11px (`text-xs2`), `tracking-wider`, `font-medium`, `rounded-full`, `px-2 py-0.5`. Live runs get the dot+`animate-pulse`; static terminal states do not.
- Impact: every page with a status indicator.
- Effort: M.

## P1 (high-impact polish)

### 8. Tighten card shadow and border discipline

- Files: `tokens-component.css`, `components/ui/card.tsx`, KpiTile, agents/skills cards
- Current: Cards use `border` (full 1px) + `shadow-sm` at rest. In dark mode the border (`hsl(217 32% 18%)`) and the card bg (`hsl(222 47% 7%)`) are close, making borders almost invisible — the cards rely on shadow that itself barely shows on dark.
- Proposed: Dark mode: drop the visible border, add `--shadow-xs` (currently unused at rest) + a 1px inner ring `inset 0 0 0 1px hsl(var(--border) / 0.5)`. Light mode: keep the border, drop the shadow at rest, add `--shadow-sm` on hover. Hover transition: `transition: box-shadow var(--duration-base) var(--ease-smooth)`.
- Impact: every card surface.
- Effort: S.

### 9. Apply tabular-nums everywhere data lives

- Files: `Home.tsx` (project table), `Insights.tsx` (Past trajectories), `RunView.tsx` (header counters), `Workers.tsx` cron column.
- Current: Body sans-serif throughout. CSS rule on line 148-151 applies tabular-nums only to `.font-mono` / `tabular-nums` classes — but the project table on `/` (Runs, Tokens, OK columns) and the Insights time column are plain `text-sm`. Numbers in adjacent rows visibly shift width.
- Proposed: Add `tabular-nums` className to every numeric cell. Also: cron expressions in workers (`*/30 * * * *`) need `font-mono text-xs2` with subtle muted color and `tracking-wide` — they currently look like accidental text.
- Impact: data legibility on Home, Insights, Workers, Project Detail.
- Effort: S.

### 10. Differentiate top-bar hierarchy + breadcrumb treatment

- Files: `apps/dashboard-web/src/components/layout/TopBar.tsx`, `Breadcrumbs.tsx`
- Current: `Mission Control` lives at the top-left of every page as a constant breadcrumb root, but it has the same weight as the per-page label. On `/projects/:id/run/:runId` the breadcrumb chain `Mission Control › csv-to-json-cli › Run 008bd6f3 › Failed pill` is flat and reads as four equal items.
- Proposed: Breadcrumbs get `text-muted-foreground` `font-medium` for intermediate steps; only the final crumb is `text-foreground font-semibold`. Add lucide `ChevronRight size-3.5 text-muted-foreground/40` separators (currently a small `›` glyph at default weight). Drop the leading "Mission Control" icon; replace with the same logomark from #4 at 16px to link home — earns its place visually.
- Impact: every route.
- Effort: S.

### 11. Run header: surface time/turns/tokens as one cohesive strip

- File: `apps/dashboard-web/src/routes/RunView.tsx` (Time/Turns/Pool progress block, ~lines 200-300 area)
- Current: Three labeled progress bars stacked in a panel + a separate "7.1k in · 2.6k out" line. The progress bars use the default `bg-muted` track with no fill color — they read as inert.
- Proposed: Replace with a single horizontal **resource bar**: `[Time 1:02/1:00:00] [Turns 5/300] [Pool 0/2]`, each segment with a thin track (`h-1 bg-muted/60`) and a fill in semantic color (time=info, turns=warning when >80%, pool=success). Tokens become an inline `7.1k in · 2.6k out` chip-pair aligned right.
- Impact: every run view.
- Effort: M.

### 12. Project list in sidebar — fix the wall-of-smoke-todos

- File: `apps/dashboard-web/src/components/layout/Sidebar.tsx` (projects section)
- Current: When project names collide ("smoke-todo" × 10), every row looks identical. No timestamp, no last-run status, no truncation+disambiguation.
- Proposed: Per-project row: 14px folder icon + project name (`text-sm`) + an 8px status dot (last run outcome — green/red/grey) right-aligned. When a row is `locked`, show a small `bg-muted text-2xs px-1` "LOCKED" inline pill. Hover shows full project name + created-at via existing `Tooltip`. Long names truncate with `truncate min-w-0` — currently they don't.
- Impact: every route (sidebar is global).
- Effort: S.

### 13. Apply motion tokens — they currently aren't used

- Files: `components/ui/button.tsx`, `card.tsx`, `dialog.tsx`, KpiTile, TaskCard
- Current: `--ease-smooth`, `--ease-spring`, `--duration-fast/quick/base` exist in tokens but most transitions are `transition` (uses Tailwind default `transition-property: all; duration: 150ms`).
- Proposed: Replace `transition`/`transition-colors`/`transition-shadow` with explicit token-driven `style={{ transition: 'background-color var(--duration-quick) var(--ease-smooth), box-shadow var(--duration-base) var(--ease-smooth)' }}` on hover-prone surfaces (buttons, cards, list rows). The motion token "Linear-style" feel only happens if you actually wire it.
- Impact: hover/press feel across the entire dashboard.
- Effort: M.

### 14. Plan canvas: dotted background + flat handle dots are amateur

- File: `apps/dashboard-web/src/components/plan/PlanCanvas.tsx`
- Current: Default `<Background />` dotted grid + default ReactFlow `Controls`. The zoom/fit controls use a generic `+`/`-`/`⛶` plain-glyph trio at bottom-left (`project-plan-dark.png`).
- Proposed: `<Background variant="dots" gap={24} size={1} color="hsl(var(--border))" />` (currently the dots are too dense and too bright). Replace `<Controls />` with a custom 3-button vertical stack using `IconButton` + lucide `ZoomIn / ZoomOut / Maximize2`, positioned bottom-right with `gap-1` and `bg-card border rounded-md p-1 shadow-xs`. The dagre layout is fine.
- Impact: `/projects/:id/plan`.
- Effort: M.

### 15. Donut + outcome panel: ditch the chart for a 0-state

- File: `apps/dashboard-web/src/components/home/OutcomeDonut.tsx`
- Current: When success=0% and only failed runs exist, the donut is a single red ring filling 100% — communicates nothing the legend doesn't already say (`home-dark.png`).
- Proposed: When total ≤ 5 runs OR a single outcome class dominates >90%, render a **stat row** instead: `<span className="size-2 rounded-full bg-destructive" /> Failed · 3 of 3 (100%)`. Render the donut only when there's an actual distribution.
- Impact: `/` empty/early-stage state.
- Effort: S.

### 16. Chat surface: differentiate user vs assistant beyond bg tint

- File: `apps/dashboard-web/src/routes/Chat.tsx`
- Current: User bubble = soft blue, assistant = plain on bg, action receipts ("Project created · csv-to-json-cli") use a green-tinted card. All three bubble types have identical radius and padding (`chat-dark.png`).
- Proposed: User bubble keeps soft-blue tint but with `rounded-2xl rounded-tr-md` (signature pinned corner). Assistant gets no fill, just `text-foreground` with the avatar+name+timestamp eyebrow. Receipt cards get a tiny lucide icon (`CheckCircle2 size-3.5 text-success`) inline with the title — they currently rely on color alone, which fails `color-not-only`.
- Impact: `/chat`.
- Effort: S.

## What NOT to do

- Don't introduce a brand accent hue. The dashboard is correctly in the "Restrained" color strategy for a power-user tool. Adding brand color would push it toward generic-SaaS.
- Don't add page-level gradients or glassmorphism. Both are absolute bans and the dashboard is correctly free of them today.
- Don't replace ReactFlow on the plan canvas — its dagre + node-renderer pattern is the right affordance. Polish the node, leave the engine.
- Don't redesign individual route information architecture (routes are sound; what's missing is rhythm, typography discipline, and removal of the stripe-accent crutch).
